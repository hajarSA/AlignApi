REST interface for the Alignment server

The Alignment server provides a REST interface for accessing the functions available on the server. Each request is an URL starting with a prefix, e.g., http://aserv.inrialpes.fr/rest/. The request type is identified by a string followed by a question mark, e.g., match?, and parameters are given in the classical URL syntax, e.g., debug=true&level=1.

The result of these requests are provided in XML. In the sequel, we describe the various request types.

find

Find alignments related to one or two ontologies.

URL: http://aserv.inrialpes.fr/rest/find?onto1=&onto2=&

Parameters:
onto1 and onto2 are ontology URIs. One of them may be not provided.

XML result :
3 fr.inrialpes.exmo.align.service.Message@1cc3436 Alignment URI ...

listmethods

Get the list of matching methods available on the server.

URL: http://aserv.inrialpes.fr/rest/listmethods?

XML result :
Class Name ...

match

Match two ontologies.

URL: http://aserv.inrialpes.fr/rest/match?onto1=&onto2=&method=&force=&

Parameters:
onto1 and onto2 are ontology URIs to be matched,
method is a matching method from the list obtained by  http://aserv.inrialpes.fr/rest/listmethods?
The parameter force is optional and  "force=off" by default,
If "force=on", a new alignment will be produced.
If "force=off", the server first tries to find an existing alignment for the two ontologies. If no alignment is found a new alignment will be produced.

XML result :
5fr.inrialpes.exmo.align.service.Message@b3869c Found alignment URI

retrieve

Retrieve an alignment in a specific format.

URL: http://aserv.inrialpes.fr/rest/retrieve?id=&method=&

Parameters:
id is URI of alignment to be retrieved
method indicates the format in which the alignment will be serialised this is a classname taken from those provided by listrenderers request.

Result :
Alignment in indicated format

trim

Trim an alignment with a threshhold.

URL: http://aserv.inrialpes.fr/rest/cut?id=&threshold=&

Parameters:
id is URI of alignment to be trimmed
threshold indicates a threshold for trimming

XML result :
Metadata in RDF

metadata

Get metadata of an alignment, i.e., avoid downloading all correspondences.

URL: http://aserv.inrialpes.fr/rest/metadata?id=

Parameters:
id is URI of alignment

XML result :
Metadata in RDF

store

Store an alignment on the server.

URL: http://aserv.inrialpes.fr/rest/store?id=

Parameters:
id is URI of alignment to be stored

XML result :
11 fr.inrialpes.exmo.align.service.Message@11dee5e Alignment URI stored

load

Upload an alignment to the server.

URI: http://aserv.inrialpes.fr/rest/load?

Since this function uses POST request method, the content of alignment does not appear in URL. Therefore, this function should be called from a Java programme.

XML result :
11 fr.inrialpes.exmo.align.service.Message@11dee5e Alignment URI loaded
